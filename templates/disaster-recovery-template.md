## Failover runbook (DR region)

**When to invoke:** primary region unavailable, SLO breach, or manual declare.

1. **Declare incident** ŌĆö note time (RPO clock), open comms channel  
2. **Verify DR readiness**
   ```bash
   gcloud run services describe {{app_slug}}-dr --region {{dr_region}} --project {{google_cloud_project}}
   ```
3. **Data** ŌĆö promote DR replica / restore latest backup *(per data platform in design contract ┬¦8)*  
4. **Traffic switch**
   - Agent Gateway / load balancer: point to DR Cloud Run URL  
   - Or DNS / global LB backend swap to `{{dr_region}}`  
5. **Validate**
   ```bash
   curl -sf "$(gcloud run services describe {{app_slug}}-dr --region {{dr_region}} --format='value(status.url)')/health"
   ```
6. **Monitor** ŌĆö Cloud Monitoring dashboards, error budget, Vertex quota in DR region  

## Failback runbook (return to primary)

1. Confirm primary region healthy  
2. Sync/replicate data primary ŌåÉ DR if needed  
3. Deploy latest revision to primary:
   ```bash
   gcloud run services update-traffic {{app_slug}} --region {{primary_region}} --to-latest
   ```
4. Shift traffic gradually (canary %) via Cloud Deploy or LB weights  
5. Scale down DR standby per strategy  
6. Post-incident review ŌĆö update golden dataset / runbook gaps
