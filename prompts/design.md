# AgentForge — Design Prompt

You are calling the AgentForge Design Agent MCP Server. The server runs validate_spec (7-signal quality gate) as Step 0 before meta-skill reasoning begins. If validate_spec returns BLOCK, report the specific signal failures and stop — do NOT proceed. If WARN, note the warnings and continue. If PASS, proceed normally. The server then produces a validated design contract with optional cosign attestation when enabled.

## Prerequisites
- MCP Server configured (see IDE Setup Guide)
- `spec.md` and `plan.md` exist in project root
- AGENTFORGE_API_KEY environment variable set (if required by your MCP server)

## Instructions
1. Read `spec.md` and `plan.md` from the project root
2. Call `design_start(spec, plan)` on the AgentForge Design Agent MCP Server
3. Receive taskId (< 2 seconds)
4. Poll `design_status(taskId)` every 3 seconds. Display progress to the developer:
   - "Loading meta-skills..." → "Reasoning about patterns..." → "Searching catalogs..."
   - "Querying Agent Registry..." → "Writing §3 STL..." → "Rendering diagram (per plan.md)..."
5. When status = "completed", call `design_result(taskId)`
6. Write returned files to workspace:
   - `design-contract.md` (PRIMARY — 15 sections with §3 Structured Topology Language)
   - If `diagram_format` is `eraserio`: `diagrams/component-topology.eraser` from `diagram_xml`, plus `diagrams/component-topology.png` from base64 `diagram_png`. Share `diagram_edit_url` with the developer if present.
   - If `diagram_format` is `drawio`: `diagrams/component-topology.drawio.xml` from `diagram_xml` + `.png`
   - If `diagram_format` is `svg`: `diagrams/component-topology.svg` from `diagram_xml` + `.png`
   - `diagrams/hadr-lifecycle.<same-format>` when returned
   ⚠ Do NOT expect `design-contract.json` — it is generated by `/validate`, not `/design`.
7. Tell the developer: "Review design-contract.md — pay attention to §3 STL tables (agents, tool bindings, data flows, sequence), §5 tool bindings, and §15 Agent Identity. Edit .md and/or diagram, then run /validate."

## MCP not configured
If the Design Agent MCP Server is unreachable, tell the developer to start the server (or check MCP settings) and retry `/design`. They may also manually author `design-contract.md` using `templates/design-contract-template.md` as a fallback.
