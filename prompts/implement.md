# AgentForge — Implement Prompt

You are generating production-ready code from a validated design contract.

## Prerequisites
- `design-contract.md` exists in project root (from Design Agent or manual authoring)
- Diagram file in `diagrams/` (from Design Agent, in format specified in plan.md)
- Skills directory: `skills/` with 6 SKILL.md files
- Skills: `skills/` with 6 SKILL.md files (each contains generation rules + `## Validation Hook` section)
- Hooks: generated by /implement from skill validation hooks into `hooks/`

## Step 0: Validate (implicit — called automatically if .json is missing or stale)
1. Check: does `design-contract.json` exist?
   - If NOT → .json hasn't been generated yet. Proceed to step 2.
   - If YES → read its `metadata.source_hash`. Compute SHA-256 of current `design-contract.md` + diagram file.
     - If hashes MATCH → .json is current. Skip to Step 1.
     - If hashes DON'T MATCH → .md or diagram was edited since last validation. Proceed to step 2.
2. Check: is the AgentForge Design Agent MCP Server configured?
   - Test: call `verify_attestation` with a dummy hash. If it responds → MCP is available.
   - If not configured → skip to Step 1 with note: "⚠️ No MCP Server — skipping validation. Run /validate when MCP is available."
3. If MCP is available: call `validate_contract(design_contract_md, diagram, format)`.
   - This runs 16 checks: 12 structural + 4 diagram↔§3 STL consistency.
   - On pass → generates .json with source_hash (and attestation when enabled).
   - If ANY BLOCK finding: report the specific finding to the developer and STOP.
     "❌ Check 13 failed: node parity. §3 STL Agents table shows 8 agents but the diagram shows 7. Fix §3 or the diagram, then re-run /implement."
   - Do NOT proceed to Step 1 until all 16 checks pass.
4. Read the VALIDATED design-contract.json (which was just generated by validate_contract).

## Step 1: Read inputs
- Read `design-contract.json` (the machine-readable contract — NOT design-contract.md)
- Read ALL files in `skills/` directory
- Each skill file contains a `## Validation Hook` section — after generating each file, run the checks listed in the hook section. Generate `hooks/` directory with enforcement scripts derived from these validation hooks.
