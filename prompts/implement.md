## Step 0: Validate (implicit — called automatically if .json is missing or stale)
1. Check: does `APP_DIR/design-contract.json` exist?
   - If NOT → .json hasn't been generated yet. Proceed to step 2.
   - If YES → read its `metadata.source_hash`. Compute SHA-256 of current `APP_DIR/design-contract.md` + diagram file.
     - If hashes MATCH → .json is current. Skip to Step 1.
     - If hashes DON'T MATCH → .md or diagram was edited since last validation. Proceed to step 2.
2. Check: is the AgentForge Design Agent MCP Server configured?
   - Test: call `verify_attestation` with a dummy hash. If it responds → MCP is available.
   - If not configured → skip to Step 1 with note: "⚠️ No MCP Server — skipping validation. Run /validate when MCP is available."
3. If MCP is available: call `validate_contract(design_contract_md, diagram, format)`.
   - This runs 16 checks: 12 structural + 4 diagram↔§3 STL consistency.
   - On pass → write `APP_DIR/design-contract.json` with source_hash (and attestation when enabled).
   - If ANY BLOCK finding: report the specific finding to the developer and STOP.
     "❌ Check 13 failed: node parity. §3 STL Agents table shows 8 agents but the diagram shows 7. Fix §3 or the diagram, then re-run /implement."
   - Do NOT proceed to Step 1 until all 16 checks pass.
4. Read the VALIDATED `APP_DIR/design-contract.json` (which was just generated by validate_contract).

## Step 1.5: Ask authentication mechanism (HARD GATE — before any code)
ADK requires the developer to choose how the agent authenticates to Gemini / Google Cloud
(same choice ADK setup asks). **Do NOT generate any app code until the user answers.**

Docs: [Gemini auth](https://adk.dev/agents/models/google-gemini/index.md) ·
[Google Cloud / Agent Platform auth](https://adk.dev/get-started/google-cloud/index.md)

1. If `APP_DIR/auth-config.yml` already exists, read it and ask:
   "I found auth-config.yml (`mechanism: …`). Keep it, or choose a different auth mechanism?"
   - Keep → proceed to Step 2 using that config.
   - Change → ask the menu below and overwrite the file.
2. If missing, **stop and ask the user** (verbatim menu):

```
Before generating code, which Google ADK auth mechanism should this app use?

(a) Google AI Studio API key — GOOGLE_API_KEY (local / quickstart)
(b) Vertex AI / Agent Platform — User credentials (ADC)
    → gcloud auth application-default login + GOOGLE_CLOUD_PROJECT + GOOGLE_CLOUD_LOCATION
(c) Vertex AI / Agent Platform — Service account
    → GOOGLE_APPLICATION_CREDENTIALS key file, or Workload Identity on Cloud Run/GKE
(d) Agent Platform Express Mode — GOOGLE_GENAI_API_KEY (rapid prototyping)

Reply with a, b, c, or d.
For (b)/(c)/(d) also provide: GCP project id and location (e.g. us-central1 or global).
```

3. Wait for the user's reply. Do not invent a default silently.
4. Write `APP_DIR/auth-config.yml` with their choice, for example:

```yaml
# Chosen at /implement Step 1.5 — drives app/auth.py and .env.example
mechanism: vertex_adc          # ai_studio_api_key | vertex_adc | service_account | express_mode
google_cloud_project: my-proj  # required for b/c/d
google_cloud_location: global  # e.g. us-central1 | global
# service_account_key_env: GOOGLE_APPLICATION_CREDENTIALS  # when mechanism=service_account
docs:
  - https://adk.dev/agents/models/google-gemini/index.md
  - https://adk.dev/get-started/google-cloud/index.md
```

5. Confirm: "✅ Auth locked: `<mechanism>`. Generating ADK code next…"
6. In Step 2 project files, emit `APP_DIR/app/auth.py` + `APP_DIR/.env.example` matching this choice
   (see `skills/adk-agents/SKILL.md` § Auth). Call `configure_auth()` from `app/agent.py` before building agents.

(`from google.adk…`). Do **not** generate LangChain, LangGraph, CrewAI, AutoGen, LlamaIndex,
